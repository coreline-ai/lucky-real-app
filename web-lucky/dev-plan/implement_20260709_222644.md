# implement_20260709_222644.md

작성 일시: `2026-07-09 22:26:44 KST`

이 문서는 이번 개발의 범위를 고정하고, 구현이 목적 밖으로 확장되지 않도록 하기 위한 작업 문서다.

## 개발 목적

모노레포에 **`web-lucky/`** 경량 웹 패키지를 두고, 1차 제품으로 **토정비결 한 해 요약** 웹앱을 구현한다.

- 사용자가 **생년월일 + 대상 연도**만 입력하면, 엔진 `Tojeong`으로 **총운 한 장 + 월별 12칸** 결과를 본다.
- 연말·신년·설 시즌 데모에 특화한다 (공유 문장, 연도 프리셋, 시즌 카피).
- 계산은 **`manseryeok-engine` 정본**을 쓰고, 개인정보는 **서버 저장 없이** 브라우저에서만 처리한다 (MVP).
- 전 화면에 **오락·자기성찰** 고지를 두어 의료·투자·법률·단정 예언으로 오해되지 않게 한다.

## 문서 기반 제약 사항

- 토정 계산 입력은 **음력 생년·월·일 + 대상 연도**. 출생 시각·성별은 토정에 사용하지 않는다.
- 양력 입력 시 반드시 음력 변환 후 엔진 호출. 변환 결과를 UI에 보여줄 것 (신뢰·디버그).
- 상괘는 **대상 연도** 납음에 의존 → 연도가 바뀌면 결과가 달라질 수 있다 (시나리오 F).
- `monthly`는 길이 12 전제. 인덱스 0 = 정월 문구 체계를 엔진 데이터 순서 그대로 사용.
- 지원 날짜 범위는 엔진 음양력/정책과 동일하게 맞추고, 밖이면 계산 전 또는 오류 UI.
- 카피: 의료·투자·법률·합격 확정·재물 보장 표현 금지. 엔진 `overall`/`monthly` 원문이 강해도 **앱 프레임 고지**로 상쇄.
- 공유 페이로드에 생년월일·음력 변환값 넣지 않음 (괘 번호·제목 일부·연도·overall 축약만).
- 오락·자기성찰 고지 없이 결과만 보여주는 UI 금지.

## 공통 진행 규칙

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
- 구현 중 발생한 이슈는 해당 Phase에서 수정하고 기록한다.
- 체크박스 상태를 실제 진행 상태와 맞게 업데이트한다.
- 문서에 없는 범위 확장은 하지 않는다.
- 요구사항이 불명확하면 가능한 케이스를 나누고 확인 후 진행한다.
- 한 기능은 유지보수 가능한 최소 책임 단위로 나누고, 각 단위의 역할과 경계를 계획에 적는다.
- 계획 밖 옵션/확장/부가 기능을 만들기 위한 분리는 하지 않는다.
- 공통화는 계획 범위 안의 중복을 줄일 때만 수행하고 과도한 추상화는 피한다.
- 기존 프로젝트 API, 공식 SDK, 표준 라이브러리를 우선 검토하고 새 의존성은 범위상 필요할 때만 포함한다.
- 데모 성공 정의: **60초 내 입력→결과**, 12칸 상호작용, 연도 변경 시 재계산, 고지 노출, 공유 문장에 생일 없음.

- 실행: `cd web-lucky && npm install && npm run dev` (또는 `build` / `preview`)
