## 개발 목적

앱 전반의 숫자 표시가 사용자에게 지나치게 낮고 부정적으로 보이는 문제를 개선한다.

현재 숫자 UX는 계산 원점수와 사용자 표시 점수가 거의 같은 값으로 노출되는 구조다. 이 때문에 시장관찰 `39`, 케미 `49점`, 오행 `0%` 같은 값이 실제 의도보다 훨씬 강한 부정 신호처럼 보인다.

이번 개발의 목적은 점수를 과장하거나 실제 투자/관계 판단처럼 보이게 만드는 것이 아니라, 오행가디언즈의 제품 원칙에 맞게 **자기점검, 관찰, 대화 소재, 수집 현황**으로 이해되는 표시 체계를 만드는 것이다.

핵심 방향:

- 시장관찰은 사용자가 직접 등록한 관심종목을 기본적으로 긍정적인 관찰 후보로 다룬다.
- 케미는 엔진 원점수를 보존하되 사용자 화면에는 부드러운 표시 점수와 해석 라벨을 보여준다.
- 오행 밸런스는 점수가 아니라 분포와 보완 포인트로 표현한다.
- 도감은 낮은 진행률이 결핍처럼 보이지 않게 수집 현황과 다음 행동을 강조한다.

## 개발 범위

- 시장관찰 `관찰 준비도` 산식과 표시 밴드를 재정의한다.
- 케미 엔진 원점수와 사용자 표시 점수를 분리한다.
- 케미 부분 점수의 `5/25`, `10/30` 같은 낮은 분수 노출을 줄이고 라벨 중심으로 바꾼다.
- 홈 오행 밸런스에서 `0%`가 결핍/불량 점수처럼 보이지 않게 문구와 시각 표현을 조정한다.
- 도감 진행률은 유지하되 `점수`처럼 보이지 않게 `수집 현황`과 보상 CTA 문구를 다듬는다.
- 공유 미리보기와 README 스크린샷에 표시되는 점수 문구도 같은 기준으로 정리한다.
- 점수/문구 안전성 테스트를 보강한다.
- S25 실기기에서 화면 캡쳐로 낮은 점수 체감이 완화됐는지 확인한다.

## 현재 진단 요약

### 시장관찰

- 현재 산식은 `riskScore + nameScore + checklistScore + tagScore` 구조다.
- 방어 모드에서 체크리스트 미완료 시 `riskScore`가 `12`로 떨어진다.
- 이름 오행 기본값은 `12`, 태그 없음은 `8`, 체크리스트 0개는 `0`이라 30~50점대가 쉽게 나온다.
- 사용자는 `관찰 준비도 39`를 `나쁜 종목` 또는 `부정 신호`처럼 받아들일 수 있다.

### 케미

- `ChemistryAnalysis.totalScore`는 TS 정본 엔진과 맞춰야 하는 원점수다.
- 현재 결과 화면과 공유 제목은 원점수를 그대로 `49점`처럼 크게 보여준다.
- 부분 점수도 `10/30`, `5/25`처럼 낮은 분수 형태로 노출된다.
- 문구는 이미 부드럽지만, 숫자가 문구보다 강하게 보인다.

### 오행 밸런스

- `percentOf`는 실제 오행 분포라 0%가 나올 수 있다.
- 사용자는 `수 0%`를 `내가 0점`처럼 읽을 수 있다.
- 이 값은 보정 대상이 아니라 표현 대상이다.

### 도감

- `owned.length/cardCatalog.length`는 수집 현황이다.
- 신규 사용자에게 낮은 숫자가 자연스럽지만, 결핍처럼 보이지 않도록 다음 행동과 무료 보상을 강조해야 한다.

## 결정 기록

- 케미 원점수는 변경하지 않는다. 엔진 parity와 기존 저장 데이터를 보존한다.
- 케미 화면/공유용 표시 점수는 별도 helper로 계산한다.
- 시장관찰은 기능 성격상 원점수 계약이 없으므로 산식을 직접 조정한다.
- 오행 밸런스와 도감은 숫자를 부풀리지 않고 문구와 보조 라벨을 개선한다.
- 숫자 밴드는 높게 보이되, 매매 추천이나 관계 단정으로 읽히지 않는 라벨을 반드시 같이 표시한다.
