## 개발 목적

추가 탭 **시장관찰** 1차 개발을 진행한다.

공식 KRX 종목 마스터를 개발 시점에 내려받아 앱 로컬 데이터로 포함하고, 사용자는 종목명을 직접 오타 입력하는 대신 검색 자동완성에서 종목을 선택한다. 선택한 종목은 로컬 관심종목으로 저장하고, 오늘의 만세력 흐름과 비교해 **투자 권유가 아닌 관찰 후보**로 정렬한다.

> 종목 검색/선택 → 관심종목 저장 → 오늘의 공격/중립/방어 모드 확인 → 오행/이름 궁합/체크리스트 기반 관찰 후보 확인

## 개발 범위

- 공식 종목 데이터 원천 검증: `금융위원회_KRX상장종목정보` 또는 KRX Open API에서 검색 마스터에 필요한 필드를 확인한다.
- 로컬 종목 마스터 생성: 개발용 스크립트가 공식 데이터를 `assets/market/kr_instruments.json` 형태로 변환한다.
- Drift 마이그레이션: 종목 마스터와 사용자 관심종목 테이블을 추가한다.
- 로컬 검색: 종목명/법인명/단축코드 기준 자동완성 검색을 제공한다.
- 관심종목 저장: 사용자가 선택한 공식 종목만 관심종목으로 저장한다.
- 이름 궁합: 종목명/법인명 발음오행을 Dart 순수 로직으로 계산한다.
- 오늘 관찰 점수: 오늘 만세력 데이터와 관심종목을 비교해 관찰 후보를 정렬한다.
- 추가 탭 UI: 하단 탭에 `시장관찰`을 추가하고 검색/관심종목/오늘 관찰 후보 화면을 제공한다.
- 안전 문구 QA: `추천`, `매수`, `매도`, `목표가`, `수익 예상` 등 투자자문처럼 보이는 표현을 금지한다.

## 공식 종목 데이터 확인 결과

`금융위원회_KRX상장종목정보` 명세상 검색 마스터에 필요한 최소 필드는 존재한다.

| 앱 필드 | 공식 데이터 필드 | 1차 사용 |
|---|---|---|
| `baseDate` | 기준일자 | 데이터 버전 표시와 갱신 판단 |
| `symbol` | 단축코드 | 종목 식별, 코드 검색 |
| `isin` | ISIN코드 | 중복 식별 보조 |
| `market` | 시장구분 | KOSPI/KOSDAQ/KONEX 등 표시 |
| `name` | 종목명 | 자동완성 검색과 표시 |
| `corpName` | 법인명 | 검색 보조와 상세 표시 |
| `corpRegistrationNo` | 법인등록번호 | 내부 중복 확인 보조, UI 기본 노출 없음 |

명세상 포함되지 않거나 1차에서 직접 얻을 수 없는 항목:

- 별칭: `삼전`, `네이버`, `현차` 같은 사용자식 축약어
- 업종/섹터/테마
- 실시간 가격, 등락률, 거래량, 시가총액
- 투자지표, 뉴스, 공시 상세
- 기업 로고와 설명문

결정:

- 1차 검색은 공식 데이터의 `종목명`, `법인명`, `단축코드`, `시장구분`으로 구현한다.
- 별칭은 1차에서 최소 수동 별칭 테이블만 허용하거나 제외한다.
- 업종/오행 테마는 완전 자동화하지 않고, 1차에서는 종목명/법인명 발음오행과 사용자가 선택한 간단 태그를 우선한다.
- 최신 시세/공시/뉴스 연결은 2차 웹 연결 과제로 분리한다.

## 1차 사용자 시나리오

1. 사용자가 하단 `시장관찰` 탭으로 이동한다.
2. 프로필이 없으면 출생정보 입력 CTA와 "시장관찰은 오락/자기점검용 참고 기능" 고지를 표시한다.
3. 프로필이 있으면 오늘의 투자 리듬을 표시한다.
   - `공격 모드`: 실행력은 좋지만 체크리스트 완료 전 판단 금지
   - `중립 모드`: 비교와 관찰에 적합
   - `방어 모드`: 충동 판단을 줄이고 기록/점검 우선
4. 사용자가 검색창에 `삼`을 입력한다.
5. 앱은 로컬 종목 마스터에서 `삼성전자`, `삼성SDI`, `삼성전기`처럼 후보를 표시한다.
6. 사용자가 종목을 선택하면 관심종목에 저장한다.
7. 관심종목 목록에는 종목명, 코드, 시장, 이름 오행 태그, 오늘 관찰 점수를 표시한다.
8. 오늘 관찰 후보는 점수순으로 정렬하되 문구는 "먼저 관찰해 볼 후보"로 제한한다.
9. 상세 화면은 점수 근거를 보여준다.
   - 오늘 리스크 모드 적합도
   - 종목명/법인명 발음오행 궁합
   - 사용자가 완료한 체크리스트
   - 사용자가 직접 지정한 관심 태그
10. 화면 하단에 "이 기능은 매수/매도 추천이 아니며 실제 투자 판단은 본인의 기준과 전문가 조언을 따르세요." 고지를 표시한다.

## 잔여 리스크 / 후속 과제

- 공식 종목 데이터 갱신 자동화는 2차 과제다. 1차는 개발자가 asset을 갱신해 배포하는 방식으로 제한한다.
- 실시간 시세/공시/뉴스는 2차에서 별도 웹 연결로 검토한다.
- 업종/섹터 기반 오행 테마는 공식 종목명 데이터만으로는 부족하므로 별도 공식 원천 또는 수동 매핑 정책이 필요하다.
- 모바일 앱에 API 키를 포함하는 구조는 피하고, 필요 시 2차에서 서버/프록시 갱신 파이프라인을 별도 계획으로 작성한다.
- 투자자문 오인 리스크는 기능명이 "시장관찰"이어도 남아 있으므로 스토어 설명, 앱 내 문구, 테스트 금지어를 함께 관리한다.
