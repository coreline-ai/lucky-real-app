## 개발 목적

시장관찰 탭 1차 구현은 공식 KRX 종목 마스터, 로컬 검색, 관심종목 저장, 오늘 리듬/체크리스트 기반 관찰 후보 표시까지 완료되어 있다.

이번 개발은 사용자가 시장관찰 탭에서 **무엇을 해야 하는지 바로 이해하지 못하는 문제**를 개선한다. 핵심은 주식 매매 판단을 제공하는 것이 아니라, 관심종목을 로컬로 정리하고 오늘의 만세력 리듬에 맞춰 **관찰 전 자기점검 흐름**을 명확하게 만드는 것이다.

## 개발 범위

- 시장관찰 첫 화면의 정보 구조를 개선한다.
- 오늘 리듬, 관찰 키워드, 체크리스트, 종목 검색, 관찰 후보의 사용 순서를 명확히 한다.
- 검색 결과와 관심종목 추가/중복/삭제 상태를 사용자가 이해하기 쉽게 만든다.
- 관찰 점수 숫자의 의미를 `관찰 준비도` 중심으로 설명한다.
- 점수 근거를 안전한 문구로 보여준다.
- 체크리스트 상태를 오늘 날짜 기준으로 로컬 보존하고 날짜가 바뀌면 새로 시작한다.
- 기존 `memo`, `userTagsJson` 필드를 활용해 관심종목별 관찰 메모/키워드 편집 UX를 추가한다.
- 투자자문 오인 가능성이 있는 문구를 방지하는 테스트를 보강한다.

## 제외 범위

- 매수, 매도, 목표가, 수익률, 상승/하락 예측, 종목 추천 기능

- 뉴스, 공시, 재무 지표 API 연동
- 서버 저장, 사용자 데이터 외부 전송, 모바일 앱 내 API 키 보관
- 해외 주식, 암호화폐, 파생상품, 펀드 상세 분석
- 업종/섹터 자동 분류의 완전 자동화
- 앱 전체 리디자인, 하단 탭 구조 변경, 신규 이미지 에셋 생성

- 기존 프로젝트 API, 공식 SDK, 표준 라이브러리를 우선 검토하고 새 의존성은 범위상 필요할 때만 포함한다.
- 시장관찰 문구는 `관찰`, `참고`, `점검`, `오늘의 리듬`, `관심종목` 중심으로 작성한다.
- 금지 표현은 UI 코드, 테스트, 스토어/문서 반영 시 함께 확인한다.

## 현재 구현 상태 요약

- `assets/market/kr_instruments.json`은 앱 번들 로컬 종목 마스터로 존재한다.
- `MarketRepository`는 마스터 import, 검색, 관심종목 추가/삭제/목록 조회를 제공한다.
- `MarketObservationScreen`은 오늘 리듬, 체크리스트, 검색, 관심종목 관찰 후보를 한 화면에 표시한다.
- `MarketChecklistController`는 오늘 KST 날짜 기준 체크리스트를 로컬 저장/복원한다.
- `MarketWatchItems.memo`, `MarketWatchItems.userTagsJson`은 관심종목 확장 카드의 `메모/키워드` 편집으로 저장한다.
- 관찰 점수는 `관찰 준비도` 진행 표시와 근거 칩으로 보여준다.
- 검색 빈 상태, 관심종목 빈 상태, 오늘 시장관찰 흐름 카드가 단계형 사용 흐름을 안내한다.

## 문서 기반 제약 사항

- 오프라인 우선이다. 종목 검색과 관찰 후보 계산은 네트워크 없이 동작해야 한다.
- 사용자 출생정보와 관심종목, 체크리스트, 메모는 기기 내부 저장소에만 보관한다.
- `features/*/presentation`에서 DB를 직접 접근하지 않는다.
- 체크리스트 보존은 기존 `MarketRepository` 또는 application/provider 경계를 통해 처리한다.
- 오늘 날짜 기준은 기존 `AppClock`/KST 흐름을 사용한다.
- 운세 결과는 실제 투자 의사결정 근거가 아니라는 고지를 유지한다.

## 확인 필요 사항 / 결정 기록

- 결정: 이번 개선은 오프라인 로컬 UX 개선으로 제한한다.
- 결정: 외부 시세/뉴스/공시 API 연결은 하지 않는다.
- 결정: 체크리스트는 날짜별 로컬 저장으로 개선한다.
- 결정: 기존 `memo`, `userTagsJson` 필드를 우선 활용하고, 불필요한 DB 마이그레이션은 피한다.
- 결정: 관찰 점수는 매매 신호가 아니라 `관찰 준비도`로 표현한다.

## 잔여 리스크 / 후속 과제

- 실제 업종/섹터 오행 매핑은 공식 종목 마스터만으로 부족하다. 별도 공식 원천 또는 수동 정책이 필요하므로 후속 과제로 둔다.
- 종목 데이터 갱신 자동화는 개발용 스크립트/배포 프로세스 과제이며 이번 UX 개선 범위가 아니다.
- 시장관찰은 기능명만으로도 투자 판단처럼 오해될 수 있으므로 앱 내 문구, 테스트, 스토어 설명을 함께 관리해야 한다.
