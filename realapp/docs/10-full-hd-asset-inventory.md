## 해상도 원칙

에셋 생성/검수 시 "Full HD 대응"을 아래처럼 고정한다.

| 유형 | 최종 앱용 해상도 | 비고 |
|---|---:|---|
| 세로 전체 화면 배경 | 1080x1920 | 스마트폰 portrait Full HD 기준 |
| 공유/SNS 템플릿 | 1080x1920 | Flutter Text 합성 공간 확보 |
| 가로/패널형 배경 | 1920x1080 | 운세/루틴/설정 패널용 |
| 2:3 카드형 에셋 | 1080x1620 | 기존 1024x1536보다 FHD 폭에 맞춘 기준 |
| 주요 수호신/심볼 컷아웃 | 2048x2048 | 투명 PNG alpha, 1024는 썸네일/임시본만 |
| 배지/작은 보상 오브젝트 | 1024x1024 | 작은 UI 표시용, alpha 필요 |
| 타일/오버레이 | 2048x2048 | 반복/애니메이션 합성용 |

생성 마스터는 가능하면 위 최종본보다 크게 만든다. 이미지젠이 최고 해상도를 직접 제공하지 못하면 최종 앱용 파생본은 반드시 위 해상도에 맞춰 저장한다.

## 다음 이미지젠 배치 권장 순서

1. P0 missing + Upgrade 대상부터 생성한다.
2. 생성 전 각 프롬프트에 목표 최종 해상도를 명시한다.
3. 이미지젠 원본을 `realapp/assets/imagegen_sources/masters/`에 저장한다.
4. 최종 앱용 파생본은 위 표의 해상도로 강제 맞춘다.
5. 투명 에셋은 chroma-key helper로 alpha 채널을 검증한다.
6. 모든 최종본을 `ASSET_MANIFEST.md`에 기록한다.
7. 검증 스크립트로 해상도, alpha 여부, 금지 문구를 확인한다.

P0 제작 후보 수:

- 신규 미제작 ID: 30개
- 기존 고해상도 업그레이드: 7개
- 카드 일러스트 P0 세트: 15개
- 오행 배경 추가 파생본: 10개
- 보조 오류/빈 상태 제안: 1개 이상

따라서 1차 앱 품질 기준으로 다음 이미지젠 작업은 최소 37개, 카드 일러스트와 오행 파생본까지 포함하면 약 62개를 목표로 잡는다.
