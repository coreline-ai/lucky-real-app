# QA / 개인정보 / 안전 명세

범위: Flutter 스마트폰 앱, 비수익화 앱 기준  
제외: 광고, 결제, 구독, IAP, 수익화 전환율, 유료 기능 QA

## 개인정보 최소수집

- [ ] 만세력 계산에 필요한 정보만 요청한다.
- [ ] 필수 입력과 선택 입력을 명확히 구분한다.
- [ ] 생년월일, 출생시간, 성별/음양 기준, 닉네임 등 수집 목적을 화면에서 설명한다.
- [ ] 이메일, 전화번호, 위치, 연락처, 사진, 광고 ID 등 불필요한 정보는 수집하지 않는다.
- [ ] 회원가입 없이 핵심 기능을 사용할 수 있다.
- [ ] 궁합/케미 기능에서 상대방 정보 입력 시 민감한 출생정보임을 안내한다.
- [ ] 테스트 계정/샘플 데이터에 실제 개인정보를 사용하지 않는다.
- [ ] 개인정보 처리방침에 “로컬 저장 중심, 서버 전송 없음 또는 제한적 전송” 정책을 명확히 쓴다.

## 로컬 저장 및 데이터 삭제

- [ ] 사용자 프로필, 생년월일, 카드/도감 보상, 루틴 기록은 로컬 저장 기준으로 설계한다.
- [ ] 로컬 DB/스토리지에 저장되는 항목 목록을 문서화한다.
- [ ] 앱 삭제 시 로컬 데이터가 함께 삭제되는지 확인한다.
- [ ] 앱 내 “내 데이터 삭제” 기능을 제공한다.
- [ ] 백업/동기화 기능이 없다면 없다고 명확히 안내한다.
- [ ] 디버그 로그에 생년월일, 출생시간, 궁합 상대 정보가 남지 않는다.
- [ ] 크래시 리포트나 분석 도구를 쓴다면 사전 고지와 최소 수집을 적용한다.
- [ ] 화면 캡처, 공유 이미지, 로그 파일에 불필요한 개인정보가 포함되지 않는다.

## 공유 전 확인

- [ ] 궁합/케미 공유 전 미리보기 화면을 제공한다.
- [ ] 공유 이미지/텍스트에 포함되는 정보가 명확히 표시된다.
- [ ] 생년월일, 출생시간, 상대방 이름/닉네임은 기본적으로 숨김 처리한다. 내 닉네임은 기본 표시하되 공유 전 숨김을 선택할 수 있다.
- [ ] 사용자가 직접 선택한 정보만 공유 결과물에 포함된다.
- [ ] 공유 버튼 직전 확인 문구를 제공한다.
- [ ] 상대방 정보가 포함될 경우 “상대방 동의 없이 공개하지 마세요” 안내를 표시한다.
- [ ] OS 공유 시트로 넘어가기 전 최종 확인이 가능하다.
- [ ] 공유 취소 후 데이터가 외부로 전송되지 않았음을 확인한다.

## 엔터테인먼트 고지

- [ ] 온보딩 또는 첫 운세 화면에 “오락/엔터테인먼트 목적” 고지를 표시한다.
- [ ] 오늘 운세, 수호신, 오행 루틴, 카드 보상은 확정적 예언처럼 표현하지 않는다.
- [ ] “반드시”, “무조건”, “실패한다”, “위험하다” 같은 단정 표현을 제한한다.
- [ ] 운세 결과는 자기성찰, 재미, 루틴 제안 중심으로 작성한다.
- [ ] 부정적 결과가 나와도 불안감을 과도하게 유발하지 않는다.
- [ ] 미성년자도 볼 수 있는 표현 수준으로 유지한다.
- [ ] 사용자가 앱 결과에 과도하게 의존하지 않도록 안내한다.

## 의료/투자/법률 회피

- [ ] 건강, 질병, 임신, 정신건강을 진단하거나 치료 조언하지 않는다.
- [ ] “병원에 가지 않아도 된다” 같은 표현을 금지한다.
- [ ] 투자 종목, 매수/매도, 대출, 보험, 세금 판단을 권유하지 않는다.
- [ ] 계약, 소송, 이혼, 상속 등 법률 판단을 지시하지 않는다.
- [ ] 관련 주제가 등장하면 전문가 상담을 권장하는 중립 문구를 사용한다.
- [ ] 운세 결과가 실제 의사결정의 근거가 아니라는 고지를 제공한다.
- [ ] 위기 상황, 자해, 폭력, 학대 관련 입력이 있을 경우 즉시 도움을 요청하라는 안전 문구를 제공한다.

## 만세력 계산 검증

- [ ] 양력/음력 입력 방식이 명확하다.
- [ ] 윤달 입력 여부를 지원하거나 미지원 시 안내한다.
- [ ] 출생시간 미상 케이스를 처리한다.
- [ ] 자시 경계, 날짜 경계, 절기 기준을 명확히 정의한다.
- [ ] 시간대 기준을 문서화한다.
- [ ] 한국 표준시 기준 계산 여부를 명확히 한다.
- [ ] 1900년대, 2000년대, 윤년, 월말, 연말, 절기 전후 테스트 케이스를 포함한다.
- [ ] 기준 만세력 자료 또는 검증된 엔진 결과와 샘플 비교표를 만든다.
- [ ] 사주 팔자, 오행 분포, 일간, 십성/관계 계산이 일관되게 재현된다.
- [ ] 계산 실패 시 임의 결과를 보여주지 않고 오류 안내를 표시한다.

## 기존 엔진 기준 QA

기존 TypeScript 엔진 기준 검증 명령 (repo 루트의 `engine/` 디렉터리에서 실행):

```bash
npm run type-check
npm run build
npm test
npm run test:engine
npm run solar-terms:verify
```

Flutter/Dart 엔진 포팅 후 추가 검증:

```bash
flutter pub get
dart format --set-exit-if-changed .
flutter analyze
flutter test
flutter test integration_test
```

엔진 경계 케이스:

- [ ] 1908-04-01 이전 입력 처리
- [ ] 2101년 이후 입력 처리
- [ ] 음력 윤달 입력
- [ ] 출생시 미상
- [ ] 자시 경계
- [ ] 야자시/조자시 정책
- [ ] 한국 법정시 전환 구간
- [ ] 존재하지 않는 민간시각
- [ ] 중복 민간시각
- [ ] 진태양시 옵션

## 접근성

- [ ] 모든 버튼에 스크린리더 라벨이 있다.
- [ ] 색상만으로 오행 상태를 구분하지 않는다.
- [ ] 오행 색상에 텍스트/아이콘/패턴 보조 정보를 제공한다.
- [ ] 다크모드와 라이트모드에서 명암비가 충분하다.
- [ ] 시스템 글자 크기 확대 시 레이아웃이 깨지지 않는다.
- [ ] 터치 타깃 크기가 충분하다.
- [ ] 카드/도감 UI가 스크린리더 순서대로 읽힌다.
- [ ] 애니메이션을 줄이는 OS 설정을 존중한다.
- [ ] 햅틱/사운드가 필수 정보 전달 수단이 되지 않는다.

## 성능 및 안정성

- [ ] 앱 첫 실행 시간이 과도하게 길지 않다.
- [ ] 오늘 운세 화면 전환이 즉시 반응한다.
- [ ] 만세력 계산이 UI 스레드를 막지 않는다.
- [ ] 카드/도감 이미지 로딩 중 플레이스홀더가 표시된다.
- [ ] 저사양 기기에서 스크롤 끊김이 심하지 않다.
- [ ] 로컬 저장 실패 시 사용자에게 복구 가능한 안내를 제공한다.
- [ ] 네트워크가 없어도 로컬 저장 데이터는 정상 조회된다.
- [ ] 배터리를 과도하게 소모하는 백그라운드 작업이 없다.
- [ ] 앱 크래시, ANR, 메모리 누수를 릴리즈 전 점검한다.
