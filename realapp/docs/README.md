# 오행가디언즈 앱 명세서

작성일: 2026-07-07  
대상: Flutter 스마트폰 앱 MVP  
원칙: 무료 앱, 오프라인 우선, 만세력 엔진 정확도 우선

## 최종 앱 정의

**오행가디언즈**는 기존 만세력 엔진의 계산 결과를 사용자가 매일 이해하고 실천할 수 있는 경험으로 바꾸는 Flutter 앱이다. 사용자는 오늘의 수호신, 오늘 운세, 오행 루틴, 카드/도감 보상, 궁합/케미 공유를 통해 자신의 하루를 가볍게 해석하고 기록한다.

한 줄 콘셉트:

> 내 사주와 오늘의 오행 흐름을 수호신, 운세, 루틴, 카드 보상, 케미 공유로 바꿔 매일 3분 즐기는 오행 라이프 RPG 앱.

## 핵심 결정

- 앱은 수익화하지 않는다.
- 광고, 구독, 인앱결제, 유료 카드팩, 유료 리포트, 유료 테마, 유료 상담 연결을 만들지 않는다.
- 과거 수익화 후보였던 상세 해석, 테마, 카드팩, 리포트는 무료 기능, 무료 보상, 무료 확장 콘텐츠로 재정의한다.
- 서버 없는 MVP를 기본으로 한다.
- 사용자 출생정보와 기록은 기기 로컬 저장을 기본값으로 한다.
- 만세력 계산은 기존 TypeScript 엔진을 기준으로 검증하고, Flutter 앱에서는 순수 Dart 엔진 포팅 또는 명확한 EngineGateway 계약으로 격리한다.
- 앱의 첫 화면은 만세력 표가 아니라 “오늘 무엇을 보면 되고 무엇을 하면 좋은지”를 보여주는 홈 화면이다.

## 기존 repo와의 관계

현재 repo의 `engine/`은 TypeScript 만세력/사주 계산 엔진 패키지다. Flutter 앱은 이 엔진의 결과를 기준으로 제품 경험을 만든다. 엔진 관련 npm 명령은 `engine/` 디렉터리에서 실행한다.

주요 기존 엔진 영역:

- `engine/src/engine/core`: 음양력, 절기, 법정시, 간지, 정규화 컨텍스트
- `engine/src/engine/saju`: 팔자, 대운/세운/월운, 십신, 격국, 용신 등
- `engine/src/engine/compatibility`: 궁합 계산
- `engine/tests/engine`: 엔진 회귀 테스트와 fixture

Flutter MVP에서 우선 필요한 엔진 결과:

- 출생 프로필 기준 사주팔자
- 오행 분포
- 십신 요약
- 오늘 일진/월운/세운
- 궁합/케미 계산에 필요한 기본 관계값
- 계산 오류 코드와 사용자 친화적 오류 변환 기준

## MVP 핵심 루프

1. 사용자가 생년월일시와 양력/음력 정보를 입력한다.
2. 앱이 만세력 엔진으로 사용자 원국과 오늘 일진을 계산한다.
3. 홈에서 오늘의 수호신, 한 줄 운세, 오행 밸런스를 보여준다.
4. 사용자는 오늘의 오행 루틴 1-3개를 확인하고 체크한다.
5. 루틴 완료 또는 일일 확인으로 무료 카드/도감 보상을 받는다.
6. 상대 프로필을 등록해 궁합/케미 결과를 보고 공유 이미지를 생성한다.
7. 하루 기록이 캘린더에 저장된다.

## 수익화 후보의 무료 기능 전환

| 기존 후보 | 최종 처리 |
|---|---|
| 프리미엄 카드팩 | 무료 시즌 카드팩, 루틴 완료/출석/도감 보상 |
| 유료 궁합 리포트 | 무료 친구/커플 케미 상세 리포트 |
| 프리미엄 오늘 운세 | 무료 심화 운세 탭 또는 주간 회고 리포트 |
| 구독형 루틴 코칭 | 무료 7일/30일 오행 루틴 챌린지 |
| 유료 테마 | 도감 달성, 시즌 이벤트, 설정에서 제공되는 무료 테마 |
| 광고 보상 카드 | 광고 없이 일일 확인/루틴 완료 보상 |
| 전문가 상담 연결 | 무료 용어 사전, 참고용 해석 가이드 |

## 완료 기준

문서 기준 MVP가 완료되려면 다음이 증명되어야 한다.

- 사용자 온보딩부터 오늘 수호신 확인까지 막힘 없이 가능하다.
- 만세력 계산 결과가 기존 엔진 fixture와 일치하거나 차이가 설명된다.
- 모든 핵심 기능은 무료로 접근 가능하다.
- 공유 기본값에서 생년월일시와 민감 메모가 숨겨진다.
- 로컬 데이터 삭제가 가능하다.
- 앱 문구는 엔터테인먼트/자기성찰 목적을 벗어나지 않는다.
- 광고/결제/구독 관련 코드, SDK, 문구, 데이터 모델이 없다.
