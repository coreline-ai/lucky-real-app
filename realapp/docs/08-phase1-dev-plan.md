## 범위 결정 원칙

1차 개발의 목표는 **한 명의 사용자가 혼자서 매일 쓰는 코어 루프를 끝까지 완성**하는 것이다.

> 온보딩 → 오늘의 수호신/운세 확인 → 루틴 1개 체크 → 일일 카드 보상 → 기록 저장

이 루프가 엔진 정확도 검증과 함께 동작하면 1차는 완료다. 관계 기능(케미/공유)과 재방문 보조 장치(알림, 테마, 스트릭 해금)는 루프가 검증된 뒤 2차에서 붙인다.

판단 기준:

- 코어 루프에 없어도 루프가 성립하면 → 2차
- 엔진 정확도/개인정보/무료 원칙에 관련되면 → 1차에서 타협하지 않음
- 06 로드맵의 Phase 1~6 중 Phase 7(케미), Phase 8의 스토어 제출은 2차로 이동

## 1차 제외 범위 (2차 이후)

| 제외 항목 | 이유 | 이동처 |
|---|---|---|
| 궁합/케미 전체 (입력, 결과, ChemistryProfile/Result) | 코어 루프에 불필요, 두 번째 프로필 UX 별도 검증 필요 | 2차 |
| 공유 이미지 생성/공유 플로우 | 케미와 함께 개인정보 게이트 검증 필요 | 2차 |
| 케미/공유 에셋 (07 문서 9, 12번 섹션) | 대응 기능이 2차 | 2차 |
| 로컬 알림 3종 | 루프 검증 전 재방문 장치 불필요 | 2차 |
| 스트릭 기반 해금, 시즌 이벤트 카드 | RoutineStreak 기록만 1차에 저장, 해금 연결은 2차 | 2차 |
| 테마 변경, 무료 테마 보상 | 기본 테마 1종으로 충분 | 2차 |
| 주간 회고, 감정 통계, 7일 흐름 요약 | 기록 데이터 축적 후 의미 | 2차 |
| 대운 표시 UI | genderMode 미선택 흐름과 함께 설계 필요, 엔진 계약에는 포함 | 2차 |
| 시간대 선택, 진태양시 보정 | Asia/Seoul 고정 (03 결정) | v2 |
| 홈 위젯, 다국어, 스토어 제출 | MVP 이후 | 백로그 |

수익화 관련 항목은 1차/2차 구분 없이 영구 제외다 (docs/README 금지 항목).

## 마일스톤

### M0. 엔진 계약 고정 — 워크스트림 A

- [ ] EngineGateway JSON 계약 문서화: `calculateFourPillars`, `calculateDailyCycle`, `calculateDailyAnalysis` (04 인터페이스 기준, `calculateChemistry`는 계약만 정의하고 구현 보류)
- [ ] 오류 모델 고정: 지원 범위 밖 날짜, 존재하지 않는/중복 민간시각, 필수 입력 누락
- [ ] `dayBoundaryRule` ↔ `midnightMode` 매핑 fixture 검증 (late_zi_hour=yaja, midnight=joja)
- [ ] 대표 생년월일 5건 + 경계 케이스(음력, 윤달, 시간 미상, 야자시, 1908/2101 범위 밖) fixture 스냅샷 생성

검증 (repo 루트의 `engine/` 디렉터리에서 실행):

```bash
npm run type-check
npm run build
npm test
npm run test:engine
npm run solar-terms:verify
```

수용 기준: fixture가 저장되고, 이후 모든 Dart 결과 비교의 기준이 된다.

### M1. Flutter 골격 + MockGateway — 워크스트림 B

- [ ] 프로젝트 구조 생성 (04 폴더 구조), `engine/`은 Flutter import 금지 검증

- [ ] drift 스키마: 04 매핑표의 1차 테이블 — `user_profiles`, `birth_profiles`, `daily_snapshots`, `daily_records`, `guardian_cards`, `owned_cards`, `routine_logs`, `routine_streaks`, `app_meta`
- [ ] MockGateway로 홈까지 화면 연결, `AppClock` 주입

검증: `flutter analyze`, `flutter test`, `dart format --set-exit-if-changed .`

### M2. 프로필 입력 + Dart 엔진 연동 — 워크스트림 A+B

### M5. QA 게이트 + 내부 베타

- [ ] 05 핵심 QA 시나리오 중 1차 범위 항목 전건 통과 (케미/공유/알림 항목 제외)
- [ ] 수익화 제거 게이트 전건 통과
- [ ] 개인정보 게이트: 로컬 저장만, 데이터 삭제 동작, 로그에 출생정보 없음
