## Runtime vs Source Policy

- Runtime assets: only explicitly listed WebP files in `realapp/pubspec.yaml`.
- Source/reference assets: PNG files under `assets/images` and `assets/imagegen_sources`.
- Imagegen working files: prompts, masters, QA reports, and optimization tools under `assets/imagegen_sources`.
- Do not register PNG originals or `assets/imagegen_sources/` wholesale in `pubspec.yaml`.
- If a new runtime image is needed, create or verify the WebP derivative first, then add that exact WebP path to `pubspec.yaml` and `AssetPaths`.
