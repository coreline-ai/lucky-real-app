# implement_20260710_132912.md

작성 일시: `2026-07-10 13:29:12 KST`

이 문서는 기존 `web-lucky`와 별개로, `mcp-server`를 실제 호출하는 "오늘 운세 브리핑 웹 앱"을 만들기 위한 개발 계획이다. 목적은 엔진 직접 사용 웹앱을 하나 더 만드는 것이 아니라, 지금까지 구축한 MCP 서버가 브라우저 제품에서 실사용 가능한지 증명하는 것이다.

## 개발 목적

`web-lucky`는 `manseryeok-engine`을 직접 import해서 동작하는 경량 웹 허브다. 이번 신규 앱은 그 구조와 분리해, 브라우저에서 MCP endpoint(`/mcp`)를 호출하고 MCP tool 응답만으로 "오늘 운세 브리핑"을 구성한다.

핵심 목표는 다음이다.

- `web-lucky`를 수정하지 않고 신규 앱 `web-mcp-daily/`를 만든다.
- 신규 앱은 `manseryeok-engine`, `../engine`, `engine/src`를 직접 import하지 않는다.
- 브라우저는 MCP JSON-RPC 흐름으로 `initialize`, `notifications/initialized`, `tools/call`을 수행한다.
- 오늘 날짜는 `Asia/Seoul` 기준으로 계산하고, 브리핑 결과에 기준 날짜를 명확히 표시한다.
- 사용자 출생 정보와 오늘 일진 정보를 MCP tool 응답의 `structuredContent` 기준으로 해석한다.
- MCP 서버가 꺼져 있거나 tool 호출이 실패하면 로컬 엔진 fallback 없이 MCP 연결 오류를 보여준다.
- 결과 화면에는 "MCP 서버를 통해 생성됨"을 명확히 드러내고, 호출 tool과 응답 크기 같은 검증 정보를 함께 표시한다.

### MCP 호출 흐름

신규 앱은 브라우저 MCP SDK 대신 `game/src/providers/mcp-client.ts`에서 검증된 최소 JSON-RPC helper 패턴을 참고한다. 단, game 코드에 직접 의존하지 않고 신규 앱 안에서 필요한 최소 책임만 구현한다.

기본 호출 순서는 다음이다.

1. `initialize`
2. `notifications/initialized`
3. `tools/call: calendar_day_info`
4. `tools/call: saju_palja`
5. `tools/call: saju_full_reading`

1차 MVP tool argument 기준:

```json
{
  "calendar_day_info": {
    "year": 2026,
    "month": 7,
    "day": 10
  },
  "saju_palja": {
    "birth": {
      "year": 1990,
      "month": 1,
      "day": 1,
      "hour": 9,
      "minute": 30,
      "gender": "unknown"
    }
  },
  "saju_full_reading": {
    "birth": {
      "year": 1990,
      "month": 1,
      "day": 1,
      "hour": 9,
      "minute": 30,
      "gender": "unknown"
    },
    "include": ["sipsin", "sinsal", "relations"]
  }
}
```

응답 크기가 커지거나 UI 요약성이 떨어질 경우 `saju_full_reading.include`는 `["sipsin"]`으로 축소한다. 1차 목표는 많은 내용을 보여주는 것이 아니라, MCP tool 응답으로 오늘 브리핑을 안정적으로 만드는 것이다.

### 사용자 입력

- 이름: 선택 입력.
- 생년월일: 필수.
- 출생 시간: 선택 입력.
- 출생 시간을 모르는 경우 hour/minute는 `null` 또는 tool schema가 허용하는 unknown 표현으로 전달한다.
- 성별: tool schema와 현재 engine 모델에 맞춰 선택/미선택을 허용한다.
- 오늘 날짜: 브라우저 로컬 시간이 아니라 `Asia/Seoul` 기준으로 계산한다.

### 결과 화면

1차 결과는 간단한 브리핑 카드 중심으로 구성한다.

- 오늘 모드
  - 예: 안정형, 확장형, 정리형, 관찰형.
  - 실제 문구는 MCP 응답의 일진/오행/십신 근거에서 생성한다.
- 오늘의 기운
  - 오늘 간지, 오행 키워드, 십신 요약.
- 나에게 들어오는 흐름
  - 개인 팔자와 오늘 일진의 관계를 한두 문장으로 요약.
- 오늘의 행동 제안
  - 생활/관계/집중/휴식 중심의 낮은 위험 조언.
  - 의료, 법률, 투자 매수/매도, 확정 예측 문구는 금지한다.
- 주의 키워드
  - 과한 기운, 피해야 할 태도, 일정 관리 포인트.
- MCP 검증 정보
  - 호출 endpoint.
  - 호출 tool 목록.
  - 각 tool 응답 byte size.
  - `structuredContent` 파싱 성공 여부.
  - 실패 시 오류 분류.

- 각 Phase는 앞선 Phase의 자체 테스트 완료 후에만 시작한다.
