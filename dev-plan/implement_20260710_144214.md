# implement_20260710_144214.md

작성 일시: `2026-07-10 14:42:14 KST`

이 문서는 MCP 연동 운세 웹앱을 `오늘 운세 브리핑`, `주간 운세`, `월간 운세`, `토정비결 연간 운세`, `대운/세운 깊이 분석`까지 확장하기 위한 개발 계획이다. 기존 `web-mcp-daily`는 오늘 운세 브리핑 데모로 검증되었으므로, 이번 계획은 그 구조를 유지하면서 MCP tool orchestration을 명확한 기능 단위로 넓힌다.

## 개발 목적

현재 `mcp-server`는 만세력·사주·토정비결·대운·달력 tool을 이미 제공한다. 이번 개발의 목적은 엔진을 직접 import하는 새 웹앱을 만드는 것이 아니라, MCP 서버의 tool surface를 실제 제품 화면에서 조합해 다섯 가지 운세 메뉴를 제공하는 것이다.

핵심 목표는 다음이다.

- `web-mcp-daily`를 MCP 운세 브리핑 앱으로 확장한다.
- 모든 계산은 `/mcp` JSON-RPC `tools/call`로 수행한다.
- `manseryeok-engine`, `../engine`, `engine/src` direct import를 금지한다.
- 오늘/주간/월간/연간/깊이 분석이 각각 어떤 MCP tool을 호출했는지 화면에 표시한다.
- 응답 크기와 호출 횟수를 관리하기 위해 compact/default 정책을 기능별로 정의한다.
- 의료, 투자, 법률, 확정 예언 표현은 금지하고 참고용 운세 브리핑으로 유지한다.

## 대상 기능

### 1. 오늘 운세 브리핑

현재 이미 1차 구현되어 있다. 이번 계획에서는 다른 탭과 일관된 메뉴 구조, 호출 로그, compact/default 정책으로 정리한다.

사용 MCP tool:

- `calendar_day_info`
- `saju_palja`
- `saju_full_reading`

기본 include:

- balanced: `["sipsin", "sinsal", "relations"]`
- compact: `["sipsin"]`

출력:

- 오늘 모드
- 오늘의 기운
- 나에게 들어오는 흐름
- 행동 제안
- 주의 키워드
- 호출 tool/응답 크기

### 2. 주간 운세

7일치 일진과 개인 사주를 조합해 주간 흐름을 만든다.

사용 MCP tool:

- `saju_palja` 1회
- `saju_full_reading` 1회
- `calendar_day_info` 7회

기본 정책:

- `saju_full_reading.include`는 기본 `["sipsin"]`으로 시작한다.
- 7일 전체에 대해 일진, 오행, 길흉, 절기 여부를 요약한다.
- 일별 긴 해석보다 "주간 흐름/좋은 날/주의 날/키워드" 중심으로 표시한다.

출력:

- 이번 주 모드
- 7일 일진 타임라인
- 집중하기 좋은 날
- 정리/주의가 필요한 날
- 주간 키워드
- 개인 일주와 주간 일진 흐름의 관계 요약

### 3. 월간 운세

월간 역학달력과 개인 사주를 조합해 월간 흐름을 만든다.

사용 MCP tool:

- `saju_palja` 1회
- `saju_full_reading` 1회
- `calendar_month` 1회, `compact=true`
- 필요 시 선택 날짜에만 `calendar_day_info`

기본 정책:

- 월 전체는 `calendar_month.compact=true`를 기본으로 한다.
- 모든 날짜의 장문 해석은 만들지 않는다.
- "월간 흐름/주요 구간/좋은 날 후보/주의 날 후보" 중심으로 표시한다.

출력:

- 이번 달 모드
- 월간 일진 분포
- 좋은 날 후보
- 주의 날 후보
- 절기/전환점
- 월간 행동 제안

### 4. 토정비결 연간 운세

토정비결을 중심으로 연간 총운과 12개월 운세를 제공한다.

사용 MCP tool:

- `tojeong_yearly` 1회
- 선택적으로 `saju_daeun` 1회
- 선택적으로 `saju_full_reading` 1회

기본 정책:

- 1차는 `tojeong_yearly`만으로 완성한다.
- 대운/사주 관점은 "심화 보기"에서 호출한다.
- 결과 문구는 토정비결 원문/요약 중심으로 유지한다.

출력:

- 올해 총운
- 토정 괘 코드/괘 번호
- 12개월 월별 운세
- 올해 주의 키워드
- 올해 활용 키워드
- 심화 보기 진입점

### 5. 대운/세운 깊이 분석

사주 원국, 대운, 세운, 월운, 용신/격국까지 더 깊게 보는 화면이다.

사용 MCP tool:

- `saju_full_reading`
- `saju_daeun`

기본 include:

- compact: `["daeun", "un"]`
- balanced: `["daeun", "un", "gyeokguk", "yongsin", "relations"]`
- expert: `["sipsin", "jijanggan", "unsung", "daeun", "un", "sinsal", "relations", "gyeokguk", "yongsin"]`

출력:

- 원국 요약
- 현재 대운
- 세운/월운
- 용신/기신
- 격국
- 관계/신살/주의 포인트
- 깊이 분석 근거

## 신규 MCP tool 추가 판단 기준

1차는 앱 orchestration으로 구현한다. 아래 조건 중 둘 이상이 확인될 때만 별도 계획으로 aggregation tool을 검토한다.

- 주간/월간에서 호출 횟수 때문에 체감 대기 시간이 과도하다.
- PlayMCP 또는 브라우저에서 response size가 반복적으로 WATCH/LARGE가 된다.
- 같은 조합 로직이 앱과 다른 클라이언트에서 중복된다.
- MCP 서버의 tool surface로 노출하는 것이 제품적으로 더 명확하다.

후보 aggregation tool:

- `fortune_daily_briefing`
- `fortune_weekly`
- `fortune_monthly`
- `fortune_yearly_tojeong`
- `fortune_deep_cycle`

이번 계획에서는 위 tool을 구현하지 않고, 필요성 검증 후 별도 계획으로 분리한다.
