## 개발 목적

기존 `mcp-server` 브라우저 대시보드를 PlayMCP 등록 전 검증에 더 직접적으로 사용할 수 있도록 보강한다. 현재 대시보드는 `/health`, `tools/list`, 20개 fixture 기반 `tools/call` smoke를 이미 제공하므로, 이번 작업은 새 앱을 만들지 않고 기존 `/dashboard`에 가능한 검증 항목을 추가한다.

핵심 목표는 다음이다.

- 대시보드에서 `X-MCP-Auth-Token` 또는 `Authorization: Bearer` 인증 헤더를 직접 입력해 `/mcp`를 호출할 수 있게 한다.
- `tools/list`와 각 `tools/call`의 응답 byte size를 브라우저에서 표시한다.
- 실패를 auth/CORS/endpoint/protocol/size/server/tool error로 분류해 PlayMCP 등록 전 원인을 빠르게 파악하게 한다.
- PlayMCP "정보 불러오기"에 해당하는 `initialize` + `tools/list` 흐름을 대시보드 상단에서 명확히 확인하게 한다.

## 개발 범위

- `mcp-server/src/dashboard.ts`
  - 등록 검증용 상단 패널 추가.
  - Auth mode 선택 UI 추가: none, `X-MCP-Auth-Token`, Bearer.
  - token 입력 필드 추가. token은 브라우저 메모리에서만 사용하고 저장하지 않는다.
  - PlayMCP 정보 불러오기 버튼과 결과 표시 영역 추가.
- `mcp-server/src/dashboard/client.ts`
  - `/mcp` 요청에 선택된 auth header 적용.
  - response text 기준 byte size 계산.
  - `tools/list` 크기와 tool call별 크기 상태 표시.
  - HTTP/JSON-RPC/timeout/error message 기반 실패 원인 분류.
  - 기존 Run All, 필터, 카드/테이블 UX 유지.
- `mcp-server/src/dashboard/styles.ts`
  - 기존 다크 대시보드 톤에 맞는 compact panel 스타일 추가.
- `mcp-server/README.md`
  - 대시보드의 auth 입력, size 표시, 오류 분류 사용법을 짧게 갱신.
- 테스트
  - 적용 전 baseline: `npm --prefix mcp-server test`, `npm --prefix mcp-server run build`.
  - 적용 후: 동일 테스트 + 브라우저 스모크로 auth 입력/size/error UI 확인.
